## Package Installation

Install the AttestProtocol SDK using your preferred package manager:

<CodeGroup>
```bash npm
npm install @attestprotocol/sdk
```

```bash pnpm
pnpm add @attestprotocol/sdk
```

```bash yarn
yarn add @attestprotocol/sdk
```

```bash bun
bun add @attestprotocol/sdk
```
</CodeGroup>

## Peer Dependencies

Install blockchain-specific dependencies based on your target chains:

<CodeGroup>
```bash Stellar
npm install @stellar/stellar-sdk
```

```bash Solana
npm install @coral-xyz/anchor @solana/web3.js
```

```bash All Chains
npm install @attestprotocol/sdk @stellar/stellar-sdk @coral-xyz/anchor @solana/web3.js
```
</CodeGroup>

<Note>
Installing all dependencies is recommended if you plan to support multiple blockchains or are unsure which chain you'll use.
</Note>

## Requirements

| Requirement | Version |
|-------------|---------|
| Node.js | >= 16.0 |
| TypeScript | >= 4.5 |
| npm/pnpm/yarn | Latest stable |

## Module Formats

The SDK supports multiple module formats for different environments:

<CodeGroup>
```typescript ES Modules
import { AttestSDK } from '@attestprotocol/sdk';
```

```javascript CommonJS
const { AttestSDK } = require('@attestprotocol/sdk');
```

```html Browser/UMD
<script src="https://unpkg.com/@attestprotocol/sdk/dist/umd/attestprotocol.js"></script>
<script>
  const sdk = AttestProtocol.AttestSDK;
</script>
```

```typescript Deno/JSR
import { AttestSDK } from 'jsr:@attestprotocol/sdk';
```
</CodeGroup>

## TypeScript Configuration

<Note>
TypeScript is recommended for better type safety and developer experience.
</Note>

Add these settings to your `tsconfig.json`:

```json tsconfig.json
{
  "compilerOptions": {
    "target": "ES2020",
    "lib": ["ES2020", "DOM"],
    "module": "ESNext",
    "moduleResolution": "node",
    "allowSyntheticDefaultImports": true,
    "esModuleInterop": true,
    "strict": true,
    "skipLibCheck": true,
    "resolveJsonModule": true
  }
}
```

## Browser Configuration

<Warning>
Browser environments require polyfills for Node.js crypto and buffer modules.
</Warning>

<CodeGroup>
```javascript Next.js
// next.config.js
/** @type {import('next').NextConfig} */
const nextConfig = {
  webpack: (config) => {
    config.resolve.fallback = {
      ...config.resolve.fallback,
      crypto: false,
      stream: false,
      buffer: false,
    };
    return config;
  },
};

module.exports = nextConfig;
```

```javascript Vite
// vite.config.js
import { defineConfig } from 'vite';
import { nodePolyfills } from 'vite-plugin-node-polyfills';

export default defineConfig({
  plugins: [nodePolyfills()],
  define: {
    global: 'globalThis',
  },
});
```

```javascript Webpack
// webpack.config.js
module.exports = {
  resolve: {
    fallback: {
      "crypto": require.resolve("crypto-browserify"),
      "stream": require.resolve("stream-browserify"),
      "buffer": require.resolve("buffer")
    }
  },
  plugins: [
    new webpack.ProvidePlugin({
      Buffer: ['buffer', 'Buffer'],
    }),
  ],
};
```
</CodeGroup>

## Verification

Verify your installation is working correctly:

```typescript
import { AttestSDK } from '@attestprotocol/sdk';

// Should log without errors
console.log('SDK loaded successfully');

// Check version
console.log('SDK version:', AttestSDK.VERSION);
```

Check installed version via CLI:

```bash
npm list @attestprotocol/sdk
```

## Build Outputs

The SDK provides multiple build outputs for different use cases:

| Format | Path | Usage |
|--------|------|-------|
| CommonJS | `dist/main/index.js` | Node.js applications |
| ES Modules | `dist/module/index.js` | Modern bundlers |
| UMD | `dist/umd/attestprotocol.js` | Browser/CDN usage |

## Troubleshooting

<AccordionGroup>
  <Accordion title="Module not found errors">
    Clear your cache and reinstall:
    ```bash
    rm -rf node_modules package-lock.json
    npm install
    ```
  </Accordion>

  <Accordion title="TypeScript compilation errors">
    - Ensure TypeScript version >= 4.5
    - Verify `tsconfig.json` settings match recommendations
    - Install Node.js types: `npm install -D @types/node`
  </Accordion>

  <Accordion title="Browser build issues">
    - Configure polyfills as shown above
    - Check for conflicting global definitions
    - Ensure bundler supports ES modules
  </Accordion>

  <Accordion title="Peer dependency warnings">
    Install missing peer dependencies:
    ```bash
    npm install @stellar/stellar-sdk @coral-xyz/anchor @solana/web3.js
    ```
  </Accordion>
</AccordionGroup>
